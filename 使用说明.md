# Mission Center 项目使用说明

## 项目概述

Mission Center 是一个任务管理应用，已创建基础项目结构。当前代码量：**467行**。

## 项目结构

```
mission-center/
├── src/
│   ├── components/        # 组件目录
│   │   └── Layout.tsx     # 布局组件
│   ├── pages/             # 页面目录
│   │   ├── TaskList.tsx   # 任务列表（基础结构）
│   │   ├── TaskDetail.tsx # 任务详情（基础结构）
│   │   └── TaskCreate.tsx # 任务创建（占位符）
│   ├── store/             # 状态管理
│   │   └── taskStore.ts   # 任务状态管理（已实现基础CRUD）
│   ├── types/             # 类型定义
│   │   └── task.ts        # 任务相关类型定义
│   ├── App.tsx            # 主应用组件（路由配置）
│   └── main.tsx           # 入口文件
├── PROMPT.md              # 给Trae模型的完整Prompt
├── TRAE规则.md            # Trae开发规则（用户体验<5分时填写）
└── package.json           # 项目配置
```

## 如何使用Prompt进行评估

### 步骤1：准备环境
1. 确保已配置PPE环境（settings.json中设置 `ppe_trae_seed_code_internal_2`）
2. 确认模型为 `Doubao-Seed-1-8-low` 或 `Doubao-Seed-1-8-minimal`

### 步骤2：打开项目
在Trae中打开 `mission-center` 项目

### 步骤3：使用Prompt和规则
1. 将 `PROMPT.md` 文件中的内容复制，作为首轮Prompt发送给Trae模型
2. 如果评估时用户体验分 < 5分，需要填写Trae规则，可以使用 `TRAE规则.md` 中的内容作为参考

### 步骤4：执行任务
让模型根据Prompt完成以下功能：
- ✅ 完整的任务创建表单
- ✅ 任务列表的完整展示和筛选
- ✅ 任务详情页的完整功能和编辑
- ✅ 可复用的UI组件
- ✅ 状态管理增强

### 步骤5：验证代码量
完成后的PR代码行数应该 > 300行（新增代码）

## Prompt特点

这个Prompt设计符合评估指南要求：

✅ **贴近日常工作**：任务管理是常见的开发需求  
✅ **适度复杂性**：包含表单、列表、详情、筛选等多个功能模块  
✅ **从0开始构建**：基于已有基础结构，需要完成核心功能  
✅ **现实意义**：类似真实工作场景中的功能开发任务  
✅ **代码量充足**：预计新增代码 > 300行

## 评估要点

使用此Prompt进行评估时，重点关注：

1. **交付完整性**：所有功能是否完整实现
2. **工具调用**：是否正确使用文件查看、代码编辑等工具
3. **错误修复**：遇到问题是否能自主修复
4. **上下文理解**：是否能理解现有代码结构和需求
5. **用户体验**：UI是否美观，交互是否流畅
6. **信心准确度**：回答是否基于证据，避免盲目自信
7. **语言一致性**：是否保持语言统一（中文）

## 预期结果

完成后的应用应该能够：
- 创建、编辑、删除任务
- 查看任务列表，支持筛选和搜索
- 查看任务详情，支持快速操作
- 所有UI组件样式美观，交互流畅

## Trae规则说明

`TRAE规则.md` 文件包含了完整的Trae开发规则，包括：
- **硬性要求**：语言规范、代码完整性要求
- **开发工作流建议**：思考先行、小步提交、自我审查等
- **代码质量要求**：组件拆分、状态管理、类型安全等
- **用户体验要求**：交互反馈、表单验证、确认操作等
- **文件组织规范**：命名规范、目录结构等

**使用场景**：
- 当评估时用户体验分 < 5分时，需要在评估表单中填写Trae规则
- 可以直接使用或参考 `TRAE规则.md` 中的内容
- 规则要围绕遇到的体验问题来写，符合日常工作场景

## 注意事项

- 确保模型使用的是 `Doubao-Seed-1-8-low` 或 `Doubao-Seed-1-8-minimal`
- 记录Session ID（首轮Seed Beta的）
- 创建PR时，标题要改为Session ID
- 确保PR代码行数 > 300行
- 如果用户体验分 < 5分，记得填写Trae规则
